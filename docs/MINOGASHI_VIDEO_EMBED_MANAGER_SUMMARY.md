# 見逃し配信 — 動画埋め込み機能の共有メモ（マネジャー向け）

**対象:** REMrealTV（mikan-kinen）トップページ  
**目的:** 前回放送の YouTube を**サイト内で視聴**できるようにし、サイト外（YouTube 本サイト）への離脱を減らす。

---

## 1. ユーザーに見えること（体験）

1. **トップページ**のヒーロー直下に「見逃し」ブロックが表示される（設定でオン/オフ可能）。
2. 最初は **YouTube のサムネイル**（16:9）が表示される。中央に再生の目印（▶）。
3. ユーザーがタップ/クリックすると、**同じ場所に埋め込みプレイヤー**に切り替わり、**自動再生**で再生が始まる。
4. 別タブで YouTube を開く必要はない（**サイト内に留まる**）。

---

## 2. 技術的にやっていること（要約）

| 項目 | 内容 |
|------|------|
| 動画の出し方 | YouTube **iframe 埋め込み**（公式の埋め込み URL） |
| 初期表示 | 軽さのため、最初は**画像（サムネ）のみ**。再生時だけ iframe を読み込む |
| サムネ画像 | YouTube が提供する `maxresdefault`（なければ `hqdefault` に自動フォールバック） |
| 自動再生 | 埋め込み URL に `autoplay=1` を付与（クリック後のユーザー操作に紐づく一般的なパターン） |

---

## 3. 視聴まわりの配慮（YouTube 側の挙動）

埋め込み URL に次のパラメータを付けて、**関連動画や「その他の動画」系の表示を抑えやすく**しています。

- **`rel=0`** … 関連動画を**同一チャンネル中心**に制限（YouTube の仕様上、完全オフは不可）
- **`playlist=（動画ID）`** … 再生リストを**1本だけ**に相当させる指定で、終了画面や一時停止時の提案を**出にくくする**目的
- **`modestbranding=1`** … YouTube ロゴ表現を控えめに

**注意:** YouTube のプレイヤー UI はサービス側のアップデートで変わるため、**常に同じ見た目を保証するものではありません**。運用上、気になる表示が出た場合は都度画面を確認してください。

---

## 4. 文言・表示の管理（運用）

- **Supabase** の `broadcast_config`（JSON）に、見逃し用の項目をまとめて保存しています。
- **管理画面**（`/admin/broadcast`、権限のある方のみ）から、表示のオン/オフ、タイトル、日付、YouTube URL/ID、説明文、**ファーストビュー導線（案A/案B）**などを変更できます。
- **コードを書き換えず**に、次回放送に合わせて差し替え可能です。

### ファーストビュー導線（どこから見逃しに気づくか）

- **案A:** ページ最上部の**カウントダウンバナー内**（REM Chat の下）に小さな朱色リンク
- **案B:** ヒーロー内「ONLINE / 無料」の下の**ピル型ボタン**

どちらもクリックで、同じ見逃しブロックまで**スムーズスクロール**します。管理画面のラジオで切り替え可能です。

**トップページに「前回の放送」をサイト内再生できるブロックを追加し、YouTube 埋め込みのパラメータで視聴体験を整え、文言と導線は管理画面と Supabase で運用できるようにした。**
